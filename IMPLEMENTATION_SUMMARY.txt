## 🚀 How It Works

### Publishing Flow

```
ADMIN CREATES CONTENT
         ↓
     Publishing
    (Status: Published/Draft)
         ↓
DATABASE STORES
         ↓
FRONTEND FETCHES
         ↓
DISPLAYS AUTOMATICALLY
    ON WEBSITE
```

```

   └─ Status: Published
```

## 📊 Database Tables

### 5 New Tables Created

```sql
1. content
   ├─ title (required)
   ├─ slug (URL-friendly)
   ├─ description
   ├─ content (HTML supported)
   ├─ featured_image
   ├─ status (draft/published)
   └─ created_at

2. gallery
   ├─ title (required)
   ├─ image_path (required)
   ├─ description
   ├─ alt_text (accessibility)
   ├─ category
   ├─ display_order
   ├─ status (active/inactive)
   └─ created_at

3. events
   ├─ title (required)
   ├─ description
   ├─ event_date (required)
   ├─ event_time
   ├─ end_date
   ├─ location
   ├─ category
   ├─ featured_image
   ├─ status (upcoming/ongoing/completed/cancelled)
   └─ created_at

4. gallery_categories (optional)
   └─ For organizing gallery images

5. event_categories (optional)
   └─ For organizing events
```
